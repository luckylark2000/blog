# 前端监控方案探索

主要是探索一些开源的前端错误监控方案，方便参考。

1. [Sentry](https://sentry.io/)
2. [rrweb](https://github.com/rrweb-io/rrweb/blob/master/guide.zh_CN.md)
3. [web-tracing](https://m-cheng-web.github.io/web-tracing/)

Sentry 比较重量级，功能也更多，使用官网的 Sass 服务的话，数据是在国外。企业私有部署的话，有一个阉割版本，且服务器需要 4 + 16G 起步，而且只允许公司内部使用，不允许打包售卖集成了 Sentry 的商业化版本软件。

如果只想轻量级使用，推荐使用 rrweb 或者 web-tracing，加上一些手动埋点应该是够用了。下面主要介绍这两个

## rrweb

是一个开源的记录页面操作的库，可以记录页面操作，主要是记录一些 event 事件。

实际使用的话基本就是，页面进行记录操作，然后发送给服务器进行保存，分析处理，回放。

一般情况下，常见上报的情况差不多是：错误上报，关键页面操作上报。场景就是遇到错误，上传遇到错误时 5-10 分钟的操作

## web-tracing

国产中文的监控库。规模相对小一些，目前 2.9k stars，功能不是很丰富，基础监控数据都是有的。

对原生 js 和 Vue2/3，react 支持度还可以。小程序不支持。
